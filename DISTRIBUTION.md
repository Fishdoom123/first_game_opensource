# Сборка .dmg для macOS и игра по сети

## Готовое приложение без .dmg (рекомендуется, если Electron не качается)

Сервер **`server.js`** теперь отдаёт игру по **HTTP на том же порту**, что и WebSocket (**8080**). Отдельный Electron для игры **не обязателен**.

### macOS — двойной щелчок

1. Установите [Node.js](https://nodejs.org/) (LTS).
2. В Терминале один раз сделайте файл исполняемым (если macOS не запускает):

   ```bash
   chmod +x start-game-mac.command
   ```

3. В Finder откройте **`start-game-mac.command`** — поднимется сервер и откроется браузер с адресом **http://127.0.0.1:8080/**.

Или вручную:

```bash
cd "/путь/к/проектик"
npm install
npm start
```

Затем в браузере откройте **http://127.0.0.1:8080/**.

### Чем поделиться с друзьями

Заархивируйте **всю папку проекта** (без `node_modules`, чтобы архив был меньше — тогда другу нужно выполнить `npm install`). Или включите `node_modules`, если хотите, чтобы у друга был только Node без `npm install`.

Для **мультиплеера в одной Wi‑Fi**: хост запускает сервер; остальные в меню вводят **`ws://IP_хоста:8080`**.

## Собрать установочный образ (.dmg, опционально)

1. Установите [Node.js](https://nodejs.org/) (LTS, 18+).
2. В папке проекта выполните:

```bash
npm install
npm run pack:mac
```

3. Готовый файл появится в каталоге **`release/`** (например `Neon Shooter-1.0.0.dmg`).

Скрипты вызывают **`npx electron-builder`**, чтобы не зависеть от глобальной установки.

### Если `npm install` падает на этапе `electron` (TLS / socket disconnected)

Часто это обрыв связи при скачивании бинарника с GitHub. Сделайте по порядку:

1. **Повторите установку** (иногда хватает одного повтора):

   ```bash
   npm install
   ```

2. **Не используйте зеркало npmmirror по умолчанию** — у части сетей ломается DNS (`lookup ... on [::1]:53: connection refused`). Сборка идёт с **официального CDN GitHub** (поведение по умолчанию после удаления `.npmrc` из проекта).

   Если GitHub у вас недоступен, попробуйте **один раз** перед `npm install`:

   ```bash
   export ELECTRON_MIRROR="https://npmmirror.com/mirrors/electron/"
   npm install
   ```

   Либо другую сеть / отключите VPN.

3. Проверьте **DNS** на Mac (Системные настройки → Сеть → DNS). Ошибка **`[::1]:53: connection refused`** часто значит, что локальный резолвер не отвечает.

4. Проверьте **VPN**, **прокси**, **файрвол**; попробуйте другую сеть.

5. Полная переустановка зависимостей:

   ```bash
   rm -rf node_modules package-lock.json
   npm install
   ```

Сообщения **`npm warn deprecated`** от вложенных пакетов `electron-builder` на сборку не влияют — их можно не трогать.

### `electron-builder: command not found`

Значит **`npm install` не дошёл до конца** (часто из‑за ошибки загрузки Electron). Сначала добейтесь успешного **`npm install`**, затем снова **`npm run pack:mac`**.

### Один .dmg и для Intel, и для Apple Silicon

Сборка на машине даёт бинарник под **архитектуру этого Mac**. Чтобы **универсальное** приложение:

```bash
npx electron-builder --mac dmg --universal
```

(дольше по времени, но один образ для всех Mac.)

### Предупреждение Gatekeeper

Без подписи Apple (`Developer ID` + нотариация) при первом запуске macOS может писать, что приложение от неизвестного разработчика. Обычно достаточно: **Системные настройки → Конфиденциальность и безопасность → Открыть всё равно**, либо один раз выполнить в Терминале (после перетаскивания `.app` в Программы):

```bash
xattr -cr "/Applications/Neon Shooter.app"
```

## Как устроен мультиплеер

- Игра подключается к серверу по **WebSocket** (`ws://` или `wss://`).
- **`npm start`** или **`start-game-mac.command`** поднимают HTTP + WebSocket на **8080**; игра открывается в браузере по **http://127.0.0.1:8080/**.
- В **Electron-сборке (.dmg)** окно открывает тот же URL (после старта сервера).
- В меню «Мультиплеер» поле **«Адрес сервера»** задаёт, куда клиент подключается (сохраняется в `localStorage`).

## Нужно ли загружать игру на сервер?

| Сценарий | Нужен ли отдельный сервер в интернете |
|----------|----------------------------------------|
| Все в **одной Wi‑Fi** сети | **Нет.** Хост запускает игру (.dmg или `node server.js`). Остальные в поле WebSocket указывают `ws://ЛОКАЛЬНЫЙ_IP_ХОСТА:8080` (например `ws://192.168.1.42:8080`). IP хоста: macOS → Системные настройки → Сеть. |
| Игра через **интернет** (разные сети) | **Да, нужен доступный извне хост.** Варианты: VPS/облако с `node server.js` и открытым портом **8080** (лучше за reverse proxy и **WSS** `wss://ваш-домен`), либо туннель (например **ngrok**) для теста. Клиенты указывают `ws://ваш-ip:8080` или `wss://...`. |

Итого: **на обычный веб-хостинг с только статическими файлами** загружать достаточно **не получится** для мультиплеера — нужен процесс Node с `server.js` (или ваш собственный WebSocket-сервер с тем же протоколом сообщений).

## Старый способ (file://)

Раньше нужно было открывать `index.html` через Live Server. Сейчас достаточно **`npm start`** и адреса **http://127.0.0.1:8080/** — скрипты и мультиплеер работают с одного процесса.

## Переменные окружения сервера (опционально)

| Переменная | Значение по умолчанию |
|------------|------------------------|
| `NEON_WS_PORT` | `8080` |
| `NEON_WS_HOST` | `0.0.0.0` (слушать все интерфейсы, удобно для LAN) |

Пример: `NEON_WS_PORT=9090 node server.js`
